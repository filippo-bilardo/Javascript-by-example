👤 Profilo Utente — ?. e ??

Seleziona un profilo utente (alcuni con dati incompleti) e osserva come ?. (optional chaining) e ?? (nullish coalescing) gestiscono proprietà mancanti senza generare errori.

Seleziona un utente

I campi in arancione mostrano il valore di default applicato da ??.
I campi in rosso sono undefined senza protezione ??.

Dati del profilo

Codice JavaScript utilizzato

// Accesso sicuro con optional chaining (?.) + default con nullish coalescing (??)
const nome = utente?.nome ?? "Ospite"
const email = utente?.email ?? "Non fornita"
const città = utente?.indirizzo?.città ?? "Non specificata"
const cap = utente?.indirizzo?.cap ?? "—"
const twitter = utente?.social?.twitter ?? "Non collegato"
const github = utente?.social?.github ?? "Non collegato"
const phone = utente?.contatti?.telefono ?? "Non disponibile"

// Senza ?. questo causerebbe TypeError se utente o indirizzo fossero null/undefined:
// const città = utente.indirizzo.città ← ERRORE se utente==null o indirizzo==undefined
